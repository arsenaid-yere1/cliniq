# Story 0.3 — Upload Clinic Logo: Implementation Plan

## Overview

Replace the "Logo upload coming soon" placeholder in the Clinic Logo settings tab with a functional upload component. The provider can upload a logo image, see a preview, and replace or remove it. The logo is stored in Supabase Storage and referenced via `logo_storage_path` in the existing `clinic_settings` table.

## Current State Analysis

**Already exists:**
- `clinic_settings.logo_storage_path` column (migration 007, nullable text)
- Settings page with 4 tabs; "Clinic Logo" tab renders placeholder text at `settings-tabs.tsx:34-36`
- `case-documents` storage bucket pattern to follow (migration 003)
- Client-side Supabase client at `lib/supabase/client.ts`
- Signed URL generation pattern in `actions/documents.ts:117-173`
- Image display pattern in `components/documents/image-preview.tsx` (plain `<img>` with signed URL)

**Needs to be built:**
- `clinic-assets` storage bucket (new migration) — shared bucket for logo + future signature (Story 0.4)
- `ClinicLogoUpload` client component with file picker, preview, replace, and remove
- Server actions: `uploadClinicLogo`, `removeClinicLogo`, `getClinicLogoUrl`
- Wire component into `settings-tabs.tsx`

### Key Discoveries:
- `logo_storage_path` column exists but is never read or written by any action — `settings-tabs.tsx:34-36`
- `updateClinicSettings` does not include `logo_storage_path` in its field set — `actions/settings.ts:19-57`
- The TUS resumable upload pattern in `lib/tus-upload.ts` is overkill for a 2MB logo — we'll use the simple `supabase.storage.upload()` API instead
- Storage RLS policies in migration 003 follow a per-bucket pattern with separate INSERT/SELECT/UPDATE/DELETE policies

## Desired End State

- The "Clinic Logo" tab shows an upload area (drag-and-drop or click-to-browse)
- After uploading, the logo preview is displayed with options to replace or remove
- If a logo already exists (from a previous session), it loads and displays on page visit
- Uploading a new logo deletes the old file from storage and updates `logo_storage_path`
- Removing a logo deletes the file from storage and nulls `logo_storage_path`
- The `clinic-assets` bucket exists with appropriate MIME types and size limits

### How to Verify:
1. Navigate to `/settings` → "Clinic Logo" tab
2. Upload a PNG/JPG/SVG image (under 2MB) — preview appears
3. Refresh the page — logo persists and preview loads
4. Upload a different logo — old one is replaced, new preview appears
5. Click remove — logo is removed, upload area reappears
6. Try uploading a file >2MB or wrong type — error is shown

## What We're NOT Doing

- No client-side crop/resize UI (post-MVP)
- No dimension validation (minimum/maximum pixel size)
- No drag-and-drop reordering or multiple logos
- No optimistic UI during upload (simple loading state is sufficient)
- No SVG sanitization (accept as-is for MVP)
- Story 0.4 (provider signature) is a separate plan — but the bucket created here will be reused

## Implementation Approach

Three phases: (1) storage bucket migration, (2) server actions, (3) UI component. Each phase builds on the previous.

```sql
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'clinic-assets',
  'clinic-assets',
  false,

    'image/jpeg',

    'image/svg+xml'
  ]
);

-- Authenticated users can upload clinic assets
CREATE POLICY "Authenticated users can upload clinic assets"
ON storage.objects FOR INSERT TO authenticated
WITH CHECK (bucket_id = 'clinic-assets');

-- Authenticated users can read clinic assets
CREATE POLICY "Authenticated users can read clinic assets"
ON storage.objects FOR SELECT TO authenticated
USING (bucket_id = 'clinic-assets');

-- Authenticated users can update clinic assets
CREATE POLICY "Authenticated users can update clinic assets"
ON storage.objects FOR UPDATE TO authenticated
USING (bucket_id = 'clinic-assets');

-- Authenticated users can delete clinic assets
CREATE POLICY "Authenticated users can delete clinic assets"
ON storage.objects FOR DELETE TO authenticated
USING (bucket_id = 'clinic-assets');
```

## Phase 2: Server Actions

### Overview
Add three server actions for logo management: upload, remove, and get signed URL. These write to the `clinic-assets` bucket and update `logo_storage_path` on the `clinic_settings` row.

### Changes Required:

#### 1. Server Actions
**File**: `src/actions/settings.ts` (existing)
**Changes**: Add `uploadClinicLogo`, `removeClinicLogo`, and `getClinicLogoUrl` functions.

```typescript
// --- Add these constants at top of file ---
const CLINIC_ASSETS_BUCKET = 'clinic-assets'
const LOGO_MAX_SIZE = 2 * 1024 * 1024 // 2 MB
const LOGO_ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/svg+xml']

// --- Add these exported functions ---

export async function uploadClinicLogo(formData: FormData) {
  const file = formData.get('file') as File | null
  if (!file) return { error: 'No file provided' }

  // Validate file type
  if (!LOGO_ALLOWED_TYPES.includes(file.type)) {
    return { error: 'Invalid file type. Please upload a JPEG, PNG, or SVG image.' }
  }

  // Validate file size
  if (file.size > LOGO_MAX_SIZE) {
    return { error: 'File too large. Maximum size is 2 MB.' }
  }

  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  // Get or create clinic settings row
  const { data: existing } = await supabase
    .from('clinic_settings')
    .select('id, logo_storage_path')
    .is('deleted_at', null)
    .maybeSingle()

  // Delete old logo from storage if it exists
  if (existing?.logo_storage_path) {
    await supabase.storage
      .from(CLINIC_ASSETS_BUCKET)
      .remove([existing.logo_storage_path])
  }

  // Generate storage path
  const ext = file.name.split('.').pop()?.toLowerCase() || 'png'
  const storagePath = `logos/${Date.now()}.${ext}`

  // Upload to storage
  const { error: uploadError } = await supabase.storage
    .from(CLINIC_ASSETS_BUCKET)
    .upload(storagePath, file, {
      contentType: file.type,
      upsert: false,
    })

  if (uploadError) return { error: uploadError.message }

  // Update clinic_settings with new path
  let result
  if (existing) {
    result = await supabase
      .from('clinic_settings')
      .update({
        logo_storage_path: storagePath,
        updated_by_user_id: user?.id,
      })
      .eq('id', existing.id)
      .select()
      .single()
  } else {
    // Edge case: no clinic_settings row yet — create one with just the logo
    result = await supabase
      .from('clinic_settings')
      .insert({
        clinic_name: '',
        logo_storage_path: storagePath,
        created_by_user_id: user?.id,
        updated_by_user_id: user?.id,
      })
      .select()
      .single()
  }

  if (result.error) return { error: result.error.message }

  revalidatePath('/settings')
  return { data: result.data }
}

export async function removeClinicLogo() {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  const { data: existing } = await supabase
    .from('clinic_settings')
    .select('id, logo_storage_path')
    .is('deleted_at', null)
    .maybeSingle()

  if (!existing?.logo_storage_path) {
    return { error: 'No logo to remove' }
  }

  // Delete from storage
  const { error: deleteError } = await supabase.storage
    .from(CLINIC_ASSETS_BUCKET)
    .remove([existing.logo_storage_path])

  if (deleteError) return { error: deleteError.message }

  // Null out the path in clinic_settings
  const { error: updateError } = await supabase
    .from('clinic_settings')
    .update({
      logo_storage_path: null,
      updated_by_user_id: user?.id,
    })
    .eq('id', existing.id)

  if (updateError) return { error: updateError.message }

  revalidatePath('/settings')
  return { success: true }
}

export async function getClinicLogoUrl() {
  const supabase = await createClient()

  const { data: settings } = await supabase
    .from('clinic_settings')
    .select('logo_storage_path')
    .is('deleted_at', null)
    .maybeSingle()

  if (!settings?.logo_storage_path) return { url: null }

  const { data, error } = await supabase.storage
    .from(CLINIC_ASSETS_BUCKET)
    .createSignedUrl(settings.logo_storage_path, 3600)

  if (error) return { error: error.message }
  return { url: data.signedUrl }
}
```

**Note on edge case**: The `uploadClinicLogo` insert with `clinic_name: ''` handles the case where a user uploads a logo before filling in clinic info. The `clinic_name` field is NOT NULL in the DB, so we use empty string. When they later fill in the Clinic Info tab, the upsert in `updateClinicSettings` will update the existing row.

## Phase 3: Upload Component & Tab Wiring

### Overview
Build the `ClinicLogoUpload` client component and replace the placeholder in `settings-tabs.tsx`.

### Changes Required:

#### 1. Logo Upload Component
**File**: `src/components/settings/clinic-logo-upload.tsx` (new)
**Changes**: Create a client component with upload, preview, replace, and remove functionality.

The component should:
- Accept `initialLogoPath` prop (from `clinicSettings.logo_storage_path`)
- On mount, if `initialLogoPath` exists, fetch signed URL via `getClinicLogoUrl()` and show preview
- Show a file input area (styled as a dashed-border drop zone) when no logo exists
- Accept only JPEG, PNG, SVG; max 2MB (validate client-side before upload)
- On file select: call `uploadClinicLogo(formData)` server action
- Show loading spinner during upload
- On success: display the uploaded image preview
- Show "Replace" and "Remove" buttons when a logo is displayed
- "Replace" opens the file picker again
- "Remove" calls `removeClinicLogo()` server action and returns to empty state
- Use `toast.success`/`toast.error` for feedback (matching existing pattern)

```typescript
'use client'

import { useState, useRef, useCallback, useEffect } from 'react'
import { Button } from '@/components/ui/button'
import { Card, CardContent, CardDescription, CardHeader, CardTitle } from '@/components/ui/card'
import { Upload, X, ImageIcon, Loader2 } from 'lucide-react'
import { toast } from 'sonner'
import { uploadClinicLogo, removeClinicLogo, getClinicLogoUrl } from '@/actions/settings'

interface ClinicLogoUploadProps {
  initialLogoPath: string | null
}

const ACCEPTED_TYPES = ['image/jpeg', 'image/png', 'image/svg+xml']
const MAX_SIZE = 2 * 1024 * 1024 // 2 MB

export function ClinicLogoUpload({ initialLogoPath }: ClinicLogoUploadProps) {
  const [logoUrl, setLogoUrl] = useState<string | null>(null)
  const [isUploading, setIsUploading] = useState(false)
  const [isRemoving, setIsRemoving] = useState(false)
  const [isLoadingPreview, setIsLoadingPreview] = useState(!!initialLogoPath)
  const fileInputRef = useRef<HTMLInputElement>(null)

  // Load existing logo on mount
  useEffect(() => {
    if (!initialLogoPath) return
    getClinicLogoUrl().then((result) => {
      if (result.url) setLogoUrl(result.url)
      setIsLoadingPreview(false)
    })
  }, [initialLogoPath])

  const handleFileSelect = useCallback(async (file: File) => {
    // Client-side validation
    if (!ACCEPTED_TYPES.includes(file.type)) {
      toast.error('Invalid file type. Please upload a JPEG, PNG, or SVG image.')
      return
    }
    if (file.size > MAX_SIZE) {
      toast.error('File too large. Maximum size is 2 MB.')
      return
    }

    setIsUploading(true)
    const formData = new FormData()
    formData.append('file', file)

    const result = await uploadClinicLogo(formData)

    if (result.error) {
      toast.error(result.error)
      setIsUploading(false)
      return
    }

    // Fetch fresh signed URL for preview
    const urlResult = await getClinicLogoUrl()
    if (urlResult.url) setLogoUrl(urlResult.url)

    toast.success('Clinic logo uploaded')
    setIsUploading(false)
  }, [])

  const handleRemove = useCallback(async () => {
    setIsRemoving(true)
    const result = await removeClinicLogo()

    if (result.error) {
      toast.error(result.error)
      setIsRemoving(false)
      return
    }

    setLogoUrl(null)
    toast.success('Clinic logo removed')
    setIsRemoving(false)
  }, [])

  const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0]
    if (file) handleFileSelect(file)
    // Reset input so the same file can be re-selected
    e.target.value = ''
  }

  const handleDrop = (e: React.DragEvent) => {
    e.preventDefault()
    const file = e.dataTransfer.files?.[0]
    if (file) handleFileSelect(file)
  }

  const handleDragOver = (e: React.DragEvent) => {
    e.preventDefault()
  }

  return (
    <Card>
      <CardHeader>
        <CardTitle>Clinic Logo</CardTitle>
        <CardDescription>
          Upload your clinic logo. It will appear on documents and invoices.
          Accepted formats: JPEG, PNG, SVG. Maximum size: 2 MB.
        </CardDescription>
      </CardHeader>
      <CardContent>
        <input
          ref={fileInputRef}
          type="file"
          accept=".jpg,.jpeg,.png,.svg"
          onChange={handleInputChange}
          className="hidden"
        />

        {isLoadingPreview ? (
          <div className="flex items-center justify-center h-48 rounded-lg border border-dashed">
            <Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />
          </div>
        ) : logoUrl ? (
          /* Preview state */
          <div className="space-y-4">
            <div className="flex items-center justify-center rounded-lg border bg-muted/30 p-6">
              {/* eslint-disable-next-line @next/next/no-img-element */}
              <img
                src={logoUrl}
                alt="Clinic logo"
                className="max-h-48 max-w-full object-contain"
              />
            </div>
            <div className="flex gap-2">
              <Button
                variant="outline"
                onClick={() => fileInputRef.current?.click()}
                disabled={isUploading}
              >
                {isUploading ? (
                  <><Loader2 className="mr-2 h-4 w-4 animate-spin" /> Uploading...</>
                ) : (
                  <><Upload className="mr-2 h-4 w-4" /> Replace</>
                )}
              </Button>
              <Button
                variant="outline"
                onClick={handleRemove}
                disabled={isRemoving}
              >
                {isRemoving ? (
                  <><Loader2 className="mr-2 h-4 w-4 animate-spin" /> Removing...</>
                ) : (
                  <><X className="mr-2 h-4 w-4" /> Remove</>
                )}
              </Button>
            </div>
          </div>
        ) : (
          /* Upload state */
          <div
            className="flex flex-col items-center justify-center h-48 rounded-lg border border-dashed cursor-pointer hover:border-primary/50 hover:bg-muted/50 transition-colors"
            onClick={() => fileInputRef.current?.click()}
            onDrop={handleDrop}
            onDragOver={handleDragOver}
          >
            {isUploading ? (
              <Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />
            ) : (
              <>
                <ImageIcon className="h-10 w-10 text-muted-foreground mb-3" />
                <p className="text-sm text-muted-foreground">
                  Click or drag and drop to upload your logo
                </p>
                <p className="text-xs text-muted-foreground mt-1">
                  JPEG, PNG, or SVG up to 2 MB
                </p>
              </>
            )}
          </div>
        )}
      </CardContent>
    </Card>
  )
}
```

## Testing Strategy

### Manual Testing Steps:
1. **Happy path**: Upload logo → see preview → refresh → logo persists
2. **Replace flow**: Upload logo → click Replace → upload different logo → verify old is gone
3. **Remove flow**: Upload logo → click Remove → verify upload area reappears
4. **Validation - type**: Try uploading a .pdf or .txt — should show error before upload
5. **Validation - size**: Try uploading a >2MB image — should show error
6. **First-run**: Clear clinic_settings (no row exists) → upload logo → verify row created
7. **Cross-tab**: Upload logo → switch to Clinic Info tab → fill info → save → switch back to Logo tab → logo still there
8. **SVG support**: Upload an SVG file — verify it displays correctly

### No Automated Tests:
This is a file upload feature with Supabase Storage dependency. Manual testing is more practical for MVP. Unit tests for the server action validation logic can be added later if desired.

## Performance Considerations

- Signed URLs are fetched on-demand (1 hour expiry) — not stored in state across tabs
- Logo images are small (max 2MB) so no lazy loading needed
- The `useEffect` fetch on mount adds one extra request but only when a logo exists
