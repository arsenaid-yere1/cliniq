# Story 3.2 — Full Template Expansion + Medical Necessity: Implementation Plan

## Overview

Expand the Initial Visit note from 8 generic sections to 15 sections that match the clinic's actual Initial Visit document template. This includes adding a "Medical Necessity" section (Story 3.2) that synthesizes MRI findings, conservative care history, persistent symptoms, and functional impairment into a PRP treatment justification narrative. The finalized view is redesigned to match the provider's existing document format.

Since no production data exists, we drop and recreate the `initial_visit_notes` table with the new schema.

## Current State Analysis

### What Exists (Story 3.1 — Complete)
- `initial_visit_notes` table with 8 TEXT section columns
- AI generation module at `src/lib/claude/generate-initial-visit.ts` with full-note and per-section regeneration
- Server actions at `src/actions/initial-visit-notes.ts` (generate, get, save, finalize, unfinalize, regenerate section, check prerequisites)
- Editor component at `src/components/clinical/initial-visit-editor.tsx` with draft/finalized/generating/failed states
- Validation schemas at `src/lib/validations/initial-visit-note.ts` (8 sections)
- Page route at `src/app/(dashboard)/patients/[caseId]/initial-visit/page.tsx`
- Sidebar entry in `src/components/patients/case-sidebar.tsx`

### What Changes
The current 8-section schema:
1. `patient_info`, `chief_complaint`, `history_of_present_illness`, `imaging_review`, `prior_treatment_summary`, `physical_exam`, `assessment`, `treatment_plan`

Becomes 15 sections matching the provider's template:
1. `introduction` — "To Whom it May Concern" opening paragraph
2. `history_of_accident` — Mechanism of injury, immediate symptoms, post-accident actions
3. `chief_complaint` — Current complaints with pain ratings
4. `past_medical_history` — PMH, surgeries, medications, allergies
5. `social_history` — Smoking/drinking, occupation
6. `review_of_systems` — General + musculoskeletal review
7. `physical_exam` — Vital signs, cervical/lumbar exam, ROM, orthopedic tests, neuro per region
8. `imaging_findings` — MRI findings by region with impressions
9. `motor_sensory_reflex` — Motor/sensory/reflex summary
10. `medical_necessity` — **NEW** Clinical correlation + PRP justification (merges "Overall Correlation" concept with medical necessity narrative)
11. `diagnoses` — ICD-10 code list
12. `treatment_plan` — PRP injection recommendations, cost estimates, conservative care, follow-up
13. `patient_education` — Education provided to patient
14. `prognosis` — Prognosis statement
15. `clinician_disclaimer` — Standard medical-legal disclaimer

### Key Discoveries
- Migration 010 exists with old schema — will be replaced in-place (no production data)
- `generate-initial-visit.ts:11-26` — system prompt needs complete rewrite to match template tone/structure
- `generate-initial-visit.ts:28-78` — tool schema needs expansion from 8 to 15 properties

**System Prompt** — key instructions based on the actual template.

**IMPORTANT — PDF-safe formatting rules** (added to fix raw markdown appearing in PDFs):
The system prompt now includes explicit formatting rules at the top that prohibit markdown syntax:
- Use `• ` (unicode bullet) instead of `- ` or `* ` for bullet points
- Use ALL CAPS sub-headings with colon (e.g., `VITAL SIGNS:`) instead of `**bold**` or `###` headings
- Use plain text list format for ROM data (e.g., `• Flexion: Normal 50° / Actual 40° / Pain: Yes`) instead of pipe tables
- No `---` horizontal rules, no `**bold**` markers

These rules ensure the AI output renders cleanly in the PDF without a markdown parser. The PDF template's `SectionBody` component also includes fallback parsing for `• ` bullets, `- ` bullets, `**bold**` markers, and ALL CAPS sub-headings to handle any residual markdown from edge cases or previously generated notes.

See the actual system prompt in `src/lib/claude/generate-initial-visit.ts`.

**Tool Definition** — `generate_initial_visit_note` tool with 15 required string properties (one per section).

**`max_tokens`**: Increase from `8192` to `16384` (15 sections of prose is substantially more content).

**Per-Section Regeneration** — `regenerateSection` function unchanged in pattern, just updated to handle new section keys and labels.

**`InitialVisitInputData` interface** — unchanged (same source data: patient, case, case summary, clinic, provider).

```tsx
            </Text>
          )}
        </View>

        <View style={styles.separator} />

        {/* Patient Info Block */}
        <View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>Patient:</Text><Text>{data.patientName}</Text></View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>DOB:</Text><Text>{data.dob}</Text></View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>Age:</Text><Text>{data.age}</Text></View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>Date of Visit:</Text><Text>{data.dateOfVisit}</Text></View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>Indication:</Text><Text>{data.indication}</Text></View>
          <View style={styles.patientInfoRow}><Text style={styles.patientLabel}>Date of Injury:</Text><Text>{data.dateOfInjury}</Text></View>
        </View>

        <View style={styles.separator} />

        {/* Introduction — special heading */}
        {data.introduction && (
          <View>
            <Text style={styles.sectionHeading}>To Whom it May Concern</Text>
            <Text style={styles.sectionBody}>{data.introduction}</Text>
          </View>
        )}

        {/* Remaining sections */}
        {sectionEntries.map(([key, label]) => {
          const content = data[key as keyof InitialVisitPdfData] as string | null
          if (!content) return null
          return (
            <View key={key} wrap={false}>
              <Text style={styles.sectionHeading}>{label}</Text>
              <Text style={styles.sectionBody}>{content}</Text>
            </View>
          )
        })}

        <View style={styles.separator} />

        {/* Signature Block */}
        <View style={styles.signatureBlock}>
          <Text style={styles.sectionBody}>Respectfully,</Text>
          {data.providerSignatureBase64 && <Image src={data.providerSignatureBase64} style={styles.signatureImage} />}
          {data.providerName && (
            <Text style={styles.providerName}>
              {data.providerName}{data.providerCredentials && `, ${data.providerCredentials}`}
            </Text>
          )}
          {data.providerNpi && <Text style={styles.providerDetail}>NPI: {data.providerNpi}</Text>}
        </View>
      </Page>
    </Document>
  )
}
```

**Key design decisions:**
- Uses `@react-pdf/renderer`'s built-in `Helvetica` font (no custom font files needed)
- `wrap={false}` on section views prevents sections from splitting across pages when possible
- Clinic logo and provider signature must be passed as **base64 strings** (not URLs) because `@react-pdf/renderer` runs on the server and cannot fetch Supabase signed URLs
- Uses `LETTER` page size (8.5" x 11") matching US medical document standard
- **Smart `SectionBody` component** handles plain-text formatting from the AI:
  - Lines starting with `• `, `- `, or `* ` are rendered as indented bullet points with proper `• ` character
  - Short ALL CAPS lines ending with `:` are rendered as bold sub-headings (e.g., `VITAL SIGNS:`, `CERVICAL SPINE EXAMINATION:`)
  - `**bold**` inline markers are parsed and rendered with `Helvetica-Bold` font
  - Double-newlines create paragraph breaks; single newlines create line breaks
  - This handles both the new plain-text AI output AND any residual markdown from older generated notes

Server-side utility that assembles data and renders the PDF to a `Buffer`:

```typescript
import { renderToBuffer } from '@react-pdf/renderer'
import { InitialVisitPdf, type InitialVisitPdfData } from './initial-visit-template'
import { createClient } from '@/lib/supabase/server'
import { format, differenceInYears } from 'date-fns'

interface RenderPdfInput {
  note: Record<string, unknown> // The initial_visit_notes row
  caseId: string
  userId: string
}

export async function renderInitialVisitPdf(input: RenderPdfInput): Promise<Buffer> {
  const supabase = await createClient()

  // Fetch case + patient data
  const { data: caseData } = await supabase
    .from('cases')
    .select('accident_date, accident_type, patient:patients!inner(first_name, last_name, date_of_birth, gender)')
    .eq('id', input.caseId)
    .is('deleted_at', null)
    .single()

  // Fetch clinic settings
  const { data: clinicSettings } = await supabase
    .from('clinic_settings')
    .select('*')
    .single()

  // Fetch provider profile
  const { data: providerProfile } = await supabase
    .from('user_profiles')
    .select('display_name, credentials, npi_number, signature_path')
    .eq('user_id', input.userId)
    .single()

  // Fetch logo as base64 (if exists)
  let clinicLogoBase64: string | undefined
  if (clinicSettings?.logo_path) {
    const { data: logoData } = await supabase.storage
      .from('clinic-assets')
      .download(clinicSettings.logo_path)
    if (logoData) {
      const buffer = Buffer.from(await logoData.arrayBuffer())
      const mime = clinicSettings.logo_path.endsWith('.png') ? 'image/png' : 'image/jpeg'
      clinicLogoBase64 = `data:${mime};base64,${buffer.toString('base64')}`
    }
  }

  // Fetch signature as base64 (if exists)
  let providerSignatureBase64: string | undefined
  if (providerProfile?.signature_path) {
    const { data: sigData } = await supabase.storage
      .from('clinic-assets')
      .download(providerProfile.signature_path)
    if (sigData) {
      const buffer = Buffer.from(await sigData.arrayBuffer())
      const mime = providerProfile.signature_path.endsWith('.png') ? 'image/png' : 'image/jpeg'
      providerSignatureBase64 = `data:${mime};base64,${buffer.toString('base64')}`
    }
  }

  // Assemble address
  const addressParts = [
    clinicSettings?.address_line1,
    clinicSettings?.address_line2,
    [clinicSettings?.city, clinicSettings?.state].filter(Boolean).join(', ') + (clinicSettings?.zip_code ? ` ${clinicSettings.zip_code}` : ''),
  ].filter(Boolean).join(', ')

  const patient = caseData?.patient as { first_name: string; last_name: string; date_of_birth: string | null; gender: string | null } | undefined
  const patientDob = patient?.date_of_birth ? new Date(patient.date_of_birth) : null

  const pdfData: InitialVisitPdfData = {
    clinicName: clinicSettings?.clinic_name || undefined,
    clinicAddress: addressParts || undefined,
    clinicPhone: clinicSettings?.phone || undefined,
    clinicFax: clinicSettings?.fax || undefined,
    clinicLogoBase64,

    patientName: patient ? `${patient.first_name} ${patient.last_name}` : 'Unknown',
    dob: patientDob ? format(patientDob, 'MM/dd/yyyy') : '—',
    age: patientDob ? differenceInYears(new Date(), patientDob) : 0,
    dateOfVisit: format(new Date(), 'MM/dd/yyyy'),
    indication: 'Pain Management Evaluation',
    dateOfInjury: caseData?.accident_date ? format(new Date(caseData.accident_date), 'MM/dd/yyyy') : '—',

    introduction: input.note.introduction as string | null,
    history_of_accident: input.note.history_of_accident as string | null,
    chief_complaint: input.note.chief_complaint as string | null,
    past_medical_history: input.note.past_medical_history as string | null,
    social_history: input.note.social_history as string | null,
    review_of_systems: input.note.review_of_systems as string | null,
    physical_exam: input.note.physical_exam as string | null,
    imaging_findings: input.note.imaging_findings as string | null,
    motor_sensory_reflex: input.note.motor_sensory_reflex as string | null,
    medical_necessity: input.note.medical_necessity as string | null,
    diagnoses: input.note.diagnoses as string | null,
    treatment_plan: input.note.treatment_plan as string | null,
    patient_education: input.note.patient_education as string | null,
    prognosis: input.note.prognosis as string | null,
    clinician_disclaimer: input.note.clinician_disclaimer as string | null,

    providerName: providerProfile?.display_name || undefined,
    providerCredentials: providerProfile?.credentials || undefined,
    providerNpi: providerProfile?.npi_number || undefined,
    providerSignatureBase64,
  }

  return renderToBuffer(<InitialVisitPdf data={pdfData} />)
}
```

## Performance Considerations

- `max_tokens` increased from `8192` to `16384` — generation may take 15-25 seconds
- 15-section form is still a single form instance — efficient re-renders
- Per-section regeneration unchanged (~2-3 seconds per section)
- Finalized view renders 15 sections — no performance concern (static text)
- PDF rendering via `@react-pdf/renderer` adds ~2-5 seconds to finalization (server-side React rendering + buffer generation)
- Logo and signature images are fetched as base64 during PDF rendering — adds minor latency but avoids URL expiration issues
