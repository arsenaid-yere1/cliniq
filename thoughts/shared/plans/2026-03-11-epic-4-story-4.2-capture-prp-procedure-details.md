# Epic 4 Story 4.2 — Capture PRP Procedure Details: Implementation Plan

## Overview

Extend the existing "Record Procedure" dialog (built in Story 4.1) with four new form sections capturing PRP-specific treatment data: PRP Preparation, Anesthesia, Injection, and Post-Procedure. All new fields are stored as new columns on the `procedures` table. The dialog becomes a single scrollable form with a sticky section nav for orientation.

---

## Current State Analysis

- `procedures` table has Story 4.1 PRP columns: `injection_site`, `laterality`, `diagnoses`, `consent_obtained`, `pain_rating`, `procedure_number` (`supabase/migrations/013_prp_procedure_encounter.sql`)
- `src/lib/validations/prp-procedure.ts` — `prpProcedureFormSchema` covers Story 4.1 fields only
- `src/actions/procedures.ts` — `createPrpProcedure` inserts Story 4.1 fields only
- `src/components/procedures/record-procedure-dialog.tsx` — two sections (Encounter Details + Vital Signs), scrollable dialog, no section nav
- Story 4.3 will consume all 4.1 + 4.2 fields for AI note generation — every column name here becomes a variable in the note template

## Desired End State

Provider opens "Record Procedure", scrolls through 6 sections (Encounter Details → PRP Preparation → Anesthesia → Injection → Post-Procedure → Vital Signs), fills in required fields, submits. All new fields persist to `procedures`. Section nav pills at the top of the dialog jump to each section on click.

### Verification:
- Submitting the form creates a `procedures` row with all Story 4.2 columns populated
- Required fields (blood draw volume, injection volume, anesthetic agent, guidance method, complications) block submission when empty
- Complications "Other – specify" reveals textarea; selecting "None" hides it and clears the value
- Section nav pills scroll the dialog to the correct section
- Existing Story 4.1 behavior is unchanged

## What We're NOT Doing

- Story 4.3 note generation — separate story
- Edit/delete of existing procedure records
- Supplies used as a structured multi-select — plain textarea is sufficient
- Guidance method as free-text — enum only (ultrasound / fluoroscopy / landmark)
- Changing the dialog to a wizard

---

## Implementation Approach

1. Migrate: add Story 4.2 columns to `procedures`
2. Extend Zod schema with 4 new sub-schemas
3. Extend `createPrpProcedure` server action to insert new fields
4. Extend dialog: add 4 new form sections + sticky section nav

## Phase 1: Database Migration

### Overview
Add Story 4.2 columns to `procedures`. All new columns are nullable (only required at the Zod layer, not DB layer) to match the existing Story 4.1 pattern.

### Changes Required:

```sql

-- ============================================

-- PRP Preparation
alter table public.procedures
  add column blood_draw_volume_ml      numeric(6,1),   -- required
  add column centrifuge_duration_min   integer,         -- optional
  add column prep_protocol             text,            -- optional (kit description)
  add column kit_lot_number            text;            -- optional

  add column anesthetic_agent          text,            -- required (e.g. "lidocaine 1%")
  add column anesthetic_dose_ml        numeric(6,1),    -- optional
  add column patient_tolerance         text check (patient_tolerance in ('tolerated_well', 'adverse_reaction'));

-- Injection
alter table public.procedures
  add column injection_volume_ml       numeric(6,1),    -- required
  add column needle_gauge              text,            -- optional (e.g. "25-gauge spinal")

-- Post-Procedure
alter table public.procedures
  add column complications             text,            -- required: "none" or free text
  add column supplies_used             text,            -- optional: free text
  add column compression_bandage       boolean,         -- optional
  add column activity_restriction_hrs  integer;         -- optional
```

## Phase 2: Extend Validation Schema

### Overview
Add four new sub-schemas to `prp-procedure.ts` and merge them into `prpProcedureFormSchema`. Required fields enforced at the Zod layer.

### Changes Required:

#### 1. Update `src/lib/validations/prp-procedure.ts`

Add sub-schemas before `prpProcedureFormSchema` and extend the top-level schema:

```ts
const prpPreparationSchema = z.object({
  blood_draw_volume_ml: z
    .number({ required_error: 'Blood draw volume is required' })
    .positive('Must be greater than 0'),
  centrifuge_duration_min: z.number().int().positive().nullable(),
  prep_protocol: z.string().optional(),
  kit_lot_number: z.string().optional(),
})

const anesthesiaSchema = z.object({
  anesthetic_agent: z.string().min(1, 'Anesthetic agent is required'),
  anesthetic_dose_ml: z.number().positive().nullable(),
  patient_tolerance: z.enum(['tolerated_well', 'adverse_reaction']).nullable(),
})

const injectionSchema = z.object({
  injection_volume_ml: z
    .number({ required_error: 'Injection volume is required' })
    .positive('Must be greater than 0'),
  needle_gauge: z.string().optional(),
  guidance_method: z.enum(['ultrasound', 'fluoroscopy', 'landmark'], {
    required_error: 'Guidance method is required',
  }),
  target_confirmed_imaging: z.boolean().nullable(),
})

const postProcedureSchema = z.object({
  complications: z.string().min(1, 'Complications field is required'),
  supplies_used: z.string().optional(),
  compression_bandage: z.boolean().nullable(),
  activity_restriction_hrs: z.number().int().positive().nullable(),
})

export const prpProcedureFormSchema = z.object({
  // --- Story 4.1 fields (unchanged) ---
  procedure_date: z.string().min(1, 'Procedure date is required'),
  injection_site: z.string().min(1, 'Injection site is required'),
  laterality: z.enum(['left', 'right', 'bilateral'], {
    required_error: 'Laterality is required',
  }),
  diagnoses: z.array(diagnosisSchema).min(1, 'At least one diagnosis is required'),
  consent_obtained: z.boolean(),
  pain_rating: z.number().int().min(0).max(10).nullable(),
  vital_signs: vitalSignsSchema,
  // --- Story 4.2 fields ---
  prp_preparation: prpPreparationSchema,
  anesthesia: anesthesiaSchema,
  injection: injectionSchema,
  post_procedure: postProcedureSchema,
})

// Export new sub-types for component use
export type PrpPreparationValues = z.infer<typeof prpPreparationSchema>
export type AnesthesiaValues = z.infer<typeof anesthesiaSchema>
export type InjectionValues = z.infer<typeof injectionSchema>
export type PostProcedureValues = z.infer<typeof postProcedureSchema>
```

Default values to add in the dialog's `useForm`:
```ts
prp_preparation: {
  blood_draw_volume_ml: null,
  centrifuge_duration_min: null,
  prep_protocol: '',
  kit_lot_number: '',
},
anesthesia: {
  anesthetic_agent: '',
  anesthetic_dose_ml: null,
  patient_tolerance: null,
},
injection: {
  injection_volume_ml: null,
  needle_gauge: '',
  guidance_method: undefined,
  target_confirmed_imaging: null,
},
post_procedure: {
  complications: '',
  supplies_used: '',
  compression_bandage: null,
  activity_restriction_hrs: null,
},
```

**Note on `blood_draw_volume_ml` / `injection_volume_ml` default**: Use `null` in defaultValues with the Zod schema using `.number({ required_error: ... })` — the numeric input pattern (`value={field.value ?? ''}`) handles `null` → `''` correctly. This matches the existing vital signs pattern.

**Note on `complications`**: The form field stores either `"none"` (when Select = "None") or free text from the textarea (when Select = "Other – specify"). The Zod schema just requires `.min(1)` — the UI enforces the correct value shape.

Each section gets an `id` attribute on its wrapping `div`. The nav pills call `scrollIntoView` on the element:

```tsx
const SECTIONS = [
  { id: 'section-encounter', label: 'Encounter' },
  { id: 'section-prp-prep', label: 'PRP Prep' },

  document.getElementById(id)?.scrollIntoView({ behavior: 'smooth', block: 'start' })
```

```tsx
<Separator />

{/* ── Post-Procedure ── */}
<div id="section-post-proc" className="space-y-4">
  <h3 className="text-sm font-semibold text-muted-foreground uppercase tracking-wide">

  {/* complications — required, Option B pattern */}
  <FormField
    control={form.control}
    name="post_procedure.complications"
    render={({ field }) => {
      const isOther = field.value !== '' && field.value !== 'none'
      return (
        <FormItem>
          <FormLabel>Complications *</FormLabel>
          <Select
            onValueChange={(val) => {
              if (val === 'none') field.onChange('none')
              else field.onChange('')  // clear to let textarea take over
            }}
            value={isOther ? 'other' : (field.value || '')}
          >
            <FormControl>
              <SelectTrigger>
                <SelectValue placeholder="Select..." />
              </SelectTrigger>
            </FormControl>
            <SelectContent>
              <SelectItem value="none">None</SelectItem>
              <SelectItem value="other">Other – specify</SelectItem>
            </SelectContent>
          </Select>
          {isOther || field.value === '' && /* after "Other" selected */ false ? null : null}
          {/* Textarea shown when "Other – specify" is selected */}
          {(isOther || (field.value === '' && /* post-select */ false)) && (
            <FormControl>
              <Textarea
                placeholder="Describe complications..."
                value={field.value === 'none' ? '' : field.value}
                onChange={(e) => field.onChange(e.target.value)}
                className="mt-2"
              />
            </FormControl>
          )}
          <FormMessage />
        </FormItem>
      )
    }}
  />
```

**Implementation note for complications**: The Select drives a local state variable `complicationsMode: 'none' | 'other' | ''` to control textarea visibility cleanly, rather than deriving it from field.value alone. This avoids the edge case where `field.value === ''` is ambiguous (form init vs. "Other" selected). Revised pattern:

```tsx
// In the component, alongside `form`:
const [complicationsMode, setComplicationsMode] = useState<'none' | 'other' | ''>('')

// In the FormField render:
<Select
  onValueChange={(val) => {
    setComplicationsMode(val as 'none' | 'other')
    if (val === 'none') {
      form.setValue('post_procedure.complications', 'none')
    } else {
      form.setValue('post_procedure.complications', '')
    }
  }}
  value={complicationsMode}
>
  <SelectContent>
    <SelectItem value="none">None</SelectItem>
    <SelectItem value="other">Other – specify</SelectItem>
  </SelectContent>
</Select>
{complicationsMode === 'other' && (
  <Textarea
    placeholder="Describe complications..."
    value={form.watch('post_procedure.complications') === 'none' ? '' : form.watch('post_procedure.complications')}
    onChange={(e) => form.setValue('post_procedure.complications', e.target.value, { shouldValidate: true })}
    className="mt-2"
  />
)}
```

Also reset `complicationsMode` to `''` in `form.reset()` call on successful submit.
