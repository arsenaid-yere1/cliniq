# Story 6.3 — Export Invoice PDF Implementation Plan

## Overview

Add a "Download PDF" button to the invoice detail page that generates an invoice PDF on-demand using `@react-pdf/renderer`. The PDF mirrors the existing HTML invoice layout (clinic header, patient/attorney/diagnoses info, line items table, payee section, notes). No storage or finalization — the PDF is generated fresh each request and streamed to the browser.

## Current State Analysis

- Invoice detail page renders a full document view in HTML at `src/components/billing/invoice-detail-client.tsx`
- Three existing PDF templates follow a consistent pattern: template component → renderer (data fetcher + buffer producer) → server action
- `@react-pdf/renderer` and `react-pdf` are already installed
- `getInvoiceWithContext` already fetches all needed data (invoice, line items, case, patient, attorney, clinic settings, provider profile)
- Logo/signature image helpers (`imageToBase64`, `getMimeType`) exist but are duplicated across renderers

### Key Discoveries:
- Invoice HTML layout: `src/components/billing/invoice-detail-client.tsx:237-359`
- Existing PDF template pattern: `src/lib/pdf/discharge-note-template.tsx`
- Existing renderer pattern: `src/lib/pdf/render-discharge-note-pdf.ts`
- Data fetching: `src/actions/billing.ts:380-418` (`getInvoiceWithContext`)
- Clinic logo fetch: `src/actions/settings.ts:224-241` (`getClinicLogoUrl`)

## Desired End State

The invoice detail page has a "Download PDF" button in the top action bar. Clicking it generates a PDF that matches the on-screen invoice layout and triggers a browser download. The PDF includes: clinic header with logo, invoice number/date/status, patient info, diagnoses, attorney info, line items table with totals, payee section, and notes.

### Verification:
- Button appears on invoice detail page next to Edit/Delete
- Clicking downloads a PDF file named `Invoice-{invoice_number}.pdf`
- PDF content matches the HTML invoice view
- Works with and without optional data (logo, attorney, notes, etc.)

## What We're NOT Doing

- No print button or print-specific CSS
- No PDF storage/finalization (no `documents` row, no Supabase Storage upload)
- No "Printable statement" export format
- No changes to invoice data model or existing functionality

## Implementation Approach

Two-phase approach: first create the PDF template and renderer, then add the server action and UI button. The template mirrors the HTML layout from `invoice-detail-client.tsx`. The renderer fetches data using the same queries as `getInvoiceWithContext` plus image downloads. A new server action returns the PDF buffer, and the client triggers a download via a Blob URL.

## Phase 2: Server Action & Download Button

### Overview
Add a server action that generates the invoice PDF on-demand, and a "Download PDF" button to the invoice detail page.

### Changes Required:

#### 1. Server Action for PDF Generation
**File**: `src/actions/billing.ts`
**Changes**: Add a new exported async function:

```ts
export async function generateInvoicePdf(invoiceId: string) {
  const { renderInvoicePdf } = await import('@/lib/pdf/render-invoice-pdf')

  try {
    const pdfBuffer = await renderInvoicePdf({ invoiceId })
    // Return as base64 string since server actions can't return Buffer directly
    return { data: Buffer.from(pdfBuffer).toString('base64') }
  } catch {
    return { error: 'Failed to generate PDF' }
  }
}
```

Uses dynamic import (same pattern as `discharge-notes.ts:460`) to avoid bundling `@react-pdf/renderer` in the client.

#### 2. Download PDF Button
**File**: `src/components/billing/invoice-detail-client.tsx`
**Changes**:

- Import `Download` from `lucide-react` and `generateInvoicePdf` from `@/actions/billing`
- Add `isGeneratingPdf` state
- Add a "Download PDF" button in the top action bar (line ~224, next to Edit/Delete):

```tsx
<Button
  variant="outline"
  size="sm"
  disabled={isGeneratingPdf}
  onClick={async () => {
    setIsGeneratingPdf(true)
    try {
      const result = await generateInvoicePdf(invoice.id)
      if (result.error) {
        toast.error(result.error)
        return
      }
      // Convert base64 to blob and trigger download
      const bytes = Uint8Array.from(atob(result.data!), c => c.charCodeAt(0))
      const blob = new Blob([bytes], { type: 'application/pdf' })
      const url = URL.createObjectURL(blob)
      const a = document.createElement('a')
      a.href = url
      a.download = `Invoice-${invoice.invoice_number}.pdf`
      a.click()
      URL.revokeObjectURL(url)
    } finally {
      setIsGeneratingPdf(false)
    }
  }}
>
  <Download className="h-4 w-4 mr-1" />
  {isGeneratingPdf ? 'Generating...' : 'Download PDF'}
</Button>
```

## Performance Considerations

- Dynamic import of `@react-pdf/renderer` avoids bundling it in the client JS
- PDF generation happens server-side; the base64 transfer to client is the main payload
- For typical invoices (< 2 pages), this should be fast (< 2 seconds)
