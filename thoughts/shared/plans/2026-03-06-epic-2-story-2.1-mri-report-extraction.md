# Epic 2, Story 2.1 — MRI Report Extraction: Implementation Plan

## Overview

Add AI-powered extraction of structured clinical data from uploaded MRI reports, plus a provider review workflow. When a user uploads an MRI PDF, the system automatically sends it to Claude API, extracts findings into a structured format, and presents them in a split-view UI where the provider can review, edit, approve, or reject the extraction.

**Multi-body-region support**: A single MRI PDF may contain reports for multiple body regions (e.g., cervical spine + lumbar spine). The extraction pipeline produces one extraction record per body region found in the document, so a single upload can generate multiple reviewable extractions.

### Key Discoveries:
- `saveDocumentMetadata()` at `src/actions/documents.ts:82` returns the inserted document row with `id` — we can use this to trigger extraction
- `getDocumentPreviewUrl()` at `src/actions/documents.ts:149` creates signed URLs for inline viewing (no download flag)
- Upload sheet processes files serially in a `for...of` loop at `src/components/documents/upload-sheet.tsx:102` — extraction call fits naturally after `saveDocumentMetadata()` succeeds
- The existing `PdfPreview` component is dialog-based (`Dialog` wrapper). We need a new inline variant for the split-view
- All server actions follow the pattern: `'use server'` → `createClient()` → Zod safeParse → DB operation → `revalidatePath()` → return `{ data }` or `{ error }`
- All tables use `uuid_generate_v4()` PKs, `created_at`/`updated_at`/`deleted_at` soft-delete, `created_by`/`updated_by` audit fields
- RLS is simple: `auth.role() = 'authenticated'` full access
- Only two env vars exist (both `NEXT_PUBLIC_`). `ANTHROPIC_API_KEY` will be the first server-only env var
- Forms use `react-hook-form` + `zodResolver` + shadcn `Form`/`FormField` components + `sonner` toasts

## Desired End State

After implementation:
1. Uploading an MRI report PDF triggers automatic extraction via Claude API
2. A single PDF with multiple body regions produces **multiple** extraction records (one per body region)
3. Extraction results are stored in `mri_extractions` table with structured data
4. The "Clinical Data" sidebar tab is enabled and links to `/patients/[caseId]/clinical`
5. The clinical data page shows a list of MRI extractions for the case
6. Clicking an extraction opens a split-view: PDF on left, editable extracted data on right
7. Provider can approve, edit + save, or reject each extraction independently
8. Provider edits are stored in `provider_overrides` JSONB, preserving original AI output
9. Extraction failures show a manual entry fallback

### How to verify:
1. Upload a PDF with document type "MRI Report" → extraction runs automatically, toast shows success with link
2. Upload a **multi-region MRI PDF** (e.g., cervical + lumbar) → multiple extractions created, toast shows count
3. Navigate to Clinical Data tab → all extractions appear in list with confidence badges
4. Click extraction → split-view shows PDF alongside extracted fields
5. Edit a field and save → `provider_overrides` contains the edit, `review_status` = `'edited'`
6. Approve an extraction → `review_status` = `'approved'`
7. Re-extract → all old records for that document soft-deleted, new extractions created

## What We're NOT Doing

- Chiropractor report extraction (Story 2.2+)
- Async extraction with realtime subscriptions (sync is fine for MVP)
- Claude Files API (beta, not ZDR-eligible)
- Production HIPAA/BAA setup (synthetic data only for now)
- Granular rate limiting on Claude API calls
- Extraction version history UI (soft-deleted records exist in DB but are not surfaced)

## Implementation Approach

Build bottom-up: database schema → extraction pipeline → server actions → UI components → page route. Each phase is independently testable.

#### 1. Claude API Extraction Module
**File**: `src/lib/claude/extract-mri.ts` (modify — currently implements single-region extraction)

**Key change**: The prompt and tool schema must support **multiple body regions** in a single PDF. Claude returns an array of `reports`, each representing one body region. The function returns `MriExtractionResult[]` instead of a single `MriExtractionResult`.

```typescript
import Anthropic from '@anthropic-ai/sdk'
import { mriExtractionResponseSchema, type MriExtractionResult } from '@/lib/validations/mri-extraction'

const anthropic = new Anthropic()

const SYSTEM_PROMPT = `You are a medical data extraction assistant for a personal injury clinic.
Extract structured information from MRI radiology reports using the provided tool.

Rules:
- A single PDF may contain MRI reports for MULTIPLE body regions (e.g., cervical + lumbar spine)
- Create a SEPARATE report object for each body region found in the document
- Extract the body region scanned (e.g., "Lumbar Spine", "Cervical Spine")
- Extract the MRI study date if present (may differ per region or be shared)
- Extract each disc level or anatomical finding individually -- do NOT combine
- For each finding, identify the spinal level (e.g., L4-L5) or anatomical location
- Include the radiologist's Impression section verbatim if present (per body region)
- If a field cannot be determined, return null -- do NOT guess
- Set confidence to "low" if document quality is poor or report is incomplete
- Add extraction_notes for anything ambiguous or missing`

const EXTRACTION_TOOL: Anthropic.Tool = {
  name: 'extract_mri_data',
  description: 'Extract structured data from one or more MRI radiology reports in a PDF',
  input_schema: {
    type: 'object',
    properties: {
      reports: {
        type: 'array',
        description: 'One report per body region found in the document. Most PDFs have 1, but multi-region PDFs will have 2+.',
        items: {
          type: 'object',
          properties: {
            body_region: {
              type: 'string',
              description: "e.g. 'Lumbar Spine', 'Cervical Spine', 'Right Knee'",
            },
            mri_date: {
              type: 'string',
              description: 'ISO 8601 date (YYYY-MM-DD) or "null" if not found',
            },
            findings: {
              type: 'array',
              items: {
                type: 'object',
                properties: {
                  level: { type: 'string', description: 'Spinal level or anatomical location' },
                  description: { type: 'string', description: 'Description of the finding' },
                  severity: {
                    type: 'string',
                    enum: ['mild', 'moderate', 'severe', 'null'],
                    description: 'Severity or "null" if not determinable',
                  },
                },
                required: ['level', 'description', 'severity'],
              },
            },
            impression_summary: {
              type: 'string',
              description: "Radiologist's impression for this body region, or \"null\"",
            },
            confidence: {
              type: 'string',
              enum: ['high', 'medium', 'low'],
            },
            extraction_notes: {
              type: 'string',
              description: 'Ambiguities or quality issues for this region. "null" if none.',
            },
          },
          required: ['body_region', 'mri_date', 'findings', 'impression_summary', 'confidence', 'extraction_notes'],
        },
      },
    },
    required: ['reports'],
  },
}

function normalizeNullString(val: unknown): string | null {
  if (val === 'null' || val === null || val === undefined) return null
  return String(val)
}

export async function extractMriFromPdf(pdfBase64: string): Promise<{
  data?: MriExtractionResult[]
  rawResponse?: unknown
  error?: string
}> {
  try {
    const response = await anthropic.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 4096,
      system: SYSTEM_PROMPT,
      tools: [EXTRACTION_TOOL],
      tool_choice: { type: 'tool', name: 'extract_mri_data' },
      messages: [{
        role: 'user',
        content: [
          {
            type: 'document',
            source: { type: 'base64', media_type: 'application/pdf', data: pdfBase64 },
          },
          { type: 'text', text: 'Extract the structured data from this MRI report now. If the document contains multiple body regions, return a separate report for each.' },
        ],
      }],
    })

    const toolBlock = response.content.find((b) => b.type === 'tool_use')
    if (!toolBlock || toolBlock.type !== 'tool_use') {
      return { error: 'No tool use response from Claude' }
    }

    const raw = toolBlock.input as Record<string, unknown>

    // Normalize "null" strings in each report
    const rawReports = Array.isArray(raw.reports) ? raw.reports : []
    const normalizedReports = rawReports.map((r: Record<string, unknown>) => ({
      body_region: r.body_region,
      mri_date: normalizeNullString(r.mri_date),
      findings: Array.isArray(r.findings)
        ? r.findings.map((f: Record<string, unknown>) => ({
            ...f,
            severity: f.severity === 'null' ? null : f.severity,
          }))
        : [],
      impression_summary: normalizeNullString(r.impression_summary),
      confidence: r.confidence,
      extraction_notes: normalizeNullString(r.extraction_notes),
    }))

    const validated = mriExtractionResponseSchema.safeParse({ reports: normalizedReports })

    if (!validated.success) {
      return { error: 'Extraction output failed validation', rawResponse: raw }
    }

    return { data: validated.data.reports, rawResponse: raw }
  } catch (err) {
    return { error: err instanceof Error ? err.message : 'Claude API call failed' }
  }
}
```

**Note on max_tokens**: Increased from 2048 to 4096 to accommodate multi-region responses.

#### 2. Server Actions
**File**: `src/actions/mri-extractions.ts` (modify — currently creates single extraction per document)

**Key change**: `extractMriReport()` now creates **multiple** extraction records when Claude returns multiple body regions. One `processing` placeholder is still created first; on success it's updated with the first report's data, and additional rows are inserted for subsequent reports.

```typescript
'use server'

import { createClient } from '@/lib/supabase/server'
import { revalidatePath } from 'next/cache'
import { extractMriFromPdf } from '@/lib/claude/extract-mri'
import type { MriExtractionResult, MriReviewFormValues } from '@/lib/validations/mri-extraction'

// --- Trigger extraction for a document ---

export async function extractMriReport(documentId: string) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  // Get document metadata
  const { data: doc, error: docError } = await supabase
    .from('documents')
    .select('id, case_id, file_path, document_type')
    .eq('id', documentId)
    .is('deleted_at', null)
    .single()

  if (docError || !doc) return { error: 'Document not found' }
  if (doc.document_type !== 'mri_report') return { error: 'Not an MRI report' }

  // Soft-delete ALL existing extractions for this document (handles multi-region re-extract)
  await supabase
    .from('mri_extractions')
    .update({ deleted_at: new Date().toISOString(), updated_by_user_id: user.id })
    .eq('document_id', documentId)
    .is('deleted_at', null)

  // Create a single placeholder extraction record (status: processing)
  const { data: extraction, error: insertError } = await supabase
    .from('mri_extractions')
    .insert({
      document_id: documentId,
      case_id: doc.case_id,
      extraction_status: 'processing',
      extraction_attempts: 1,
      created_by_user_id: user.id,
      updated_by_user_id: user.id,
    })
    .select()
    .single()

  if (insertError || !extraction) return { error: 'Failed to create extraction record' }

  // Download PDF from Supabase Storage
  const { data: fileData, error: downloadError } = await supabase.storage
    .from('case-documents')
    .download(doc.file_path)

  if (downloadError || !fileData) {
    await supabase.from('mri_extractions').update({
      extraction_status: 'failed',
      extraction_error: 'Failed to download PDF from storage',
      updated_by_user_id: user.id,
    }).eq('id', extraction.id)
    return { error: 'Failed to download PDF' }
  }

  // Check for encrypted PDF
  const firstBytes = new Uint8Array(await fileData.slice(0, 8192).arrayBuffer())
  const headerStr = new TextDecoder().decode(firstBytes)
  if (headerStr.includes('/Encrypt')) {
    await supabase.from('mri_extractions').update({
      extraction_status: 'failed',
      extraction_error: 'PDF is encrypted/password-protected',
      updated_by_user_id: user.id,
    }).eq('id', extraction.id)
    return { error: 'PDF is encrypted. Please upload an unprotected version.' }
  }

  // Convert to base64 and call Claude
  const arrayBuffer = await fileData.arrayBuffer()
  const pdfBase64 = Buffer.from(arrayBuffer).toString('base64')

  const result = await extractMriFromPdf(pdfBase64)

  if (result.error || !result.data?.length) {
    // Retry once on failure
    const retry = await extractMriFromPdf(pdfBase64)

    if (retry.error || !retry.data?.length) {
      await supabase.from('mri_extractions').update({
        extraction_status: 'failed',
        extraction_error: retry.error ?? result.error ?? 'Extraction failed',
        extraction_attempts: 2,
        raw_ai_response: retry.rawResponse ?? result.rawResponse ?? null,
        updated_by_user_id: user.id,
      }).eq('id', extraction.id)
      revalidatePath(`/patients/${doc.case_id}/clinical`)
      return { error: retry.error ?? result.error }
    }

    // Retry succeeded
    const ids = await insertMultiRegionExtractions(
      supabase, extraction.id, documentId, doc.case_id, retry, user.id, 2,
    )
    revalidatePath(`/patients/${doc.case_id}/clinical`)
    return { data: { extractionIds: ids } }
  }

  // First attempt succeeded
  const ids = await insertMultiRegionExtractions(
    supabase, extraction.id, documentId, doc.case_id, result, user.id, 1,
  )
  revalidatePath(`/patients/${doc.case_id}/clinical`)
  revalidatePath(`/patients/${doc.case_id}/documents`)
  return { data: { extractionIds: ids } }
}

/**
 * Creates one extraction record per body region.
 * - First report updates the existing placeholder row (avoids orphan on failure)
 * - Additional reports are inserted as new rows
 * - All share the same document_id and raw_ai_response
 */
async function insertMultiRegionExtractions(
  supabase: Awaited<ReturnType<typeof createClient>>,
  placeholderId: string,
  documentId: string,
  caseId: string,
  result: { data?: MriExtractionResult[]; rawResponse?: unknown },
  userId: string,
  attempts: number,
): Promise<string[]> {
  const reports = result.data!
  const ids: string[] = []
  const now = new Date().toISOString()

  for (let i = 0; i < reports.length; i++) {
    const report = reports[i]
    const fields = {
      extraction_status: 'completed' as const,
      body_region: report.body_region,
      mri_date: report.mri_date,
      findings: report.findings,
      impression_summary: report.impression_summary,
      ai_model: 'claude-sonnet-4-6',
      ai_confidence: report.confidence,
      extraction_notes: report.extraction_notes,
      raw_ai_response: result.rawResponse ?? null,
      extraction_attempts: attempts,
      extracted_at: now,
      updated_by_user_id: userId,
    }

    if (i === 0) {
      // Update the placeholder row with first report
      await supabase.from('mri_extractions').update(fields).eq('id', placeholderId)
      ids.push(placeholderId)
    } else {
      // Insert additional rows for subsequent body regions
      const { data } = await supabase.from('mri_extractions').insert({
        ...fields,
        document_id: documentId,
        case_id: caseId,
        created_by_user_id: userId,
      }).select('id').single()
      if (data) ids.push(data.id)
    }
  }

  return ids
}
```

### Edge Cases to Test:
- Very large MRI report (close to 32MB) — should still work within Claude limits
- Multi-region MRI with 3+ body regions — all regions extracted as separate records
- MRI report with no impression section — `impression_summary` should be null
- MRI report with no clear date — `mri_date` should be null
- Multi-region MRI where regions have different dates — each extraction stores its own date
- Poor quality scan — confidence should be "low"
- Claude API timeout — retry should kick in, then fail gracefully

## Performance Considerations

- Extraction is synchronous (~5-15 seconds). Acceptable for MVP with loading spinner.
- PDF download from Supabase Storage is server-to-server (fast).
- Base64 encoding doubles file size in memory — for a 10MB PDF, ~20MB RAM. Fine for serverless.
- Signed URLs expire in 1 hour — sufficient for a review session.

## Migration Notes

- No data migration needed — this is a new table
- The `schema_version` column (default 1) enables future schema evolution without breaking existing data
- If the extraction prompt/schema changes later, bump `schema_version` and handle both versions in the review UI
