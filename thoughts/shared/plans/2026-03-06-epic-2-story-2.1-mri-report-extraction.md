# Epic 2, Story 2.1 — MRI Report Extraction: Implementation Plan

## Overview

Add AI-powered extraction of structured clinical data from uploaded MRI reports, plus a provider review workflow. When a user uploads an MRI PDF, the system automatically sends it to Claude API, extracts findings into a structured format, and presents them in a split-view UI where the provider can review, edit, approve, or reject the extraction.

## Current State Analysis

The codebase already has a complete document upload pipeline:

| Component | Location |
|-----------|----------|
| TUS resumable upload | `src/components/documents/upload-sheet.tsx`, `src/lib/tus-upload.ts` |
| Document metadata & server actions | `src/actions/documents.ts` |
| PDF preview (dialog-based) | `src/components/documents/pdf-preview.tsx` |
| Document list with type filters | `src/components/documents/document-list.tsx` |
| `mri_report` document type | `src/lib/validations/document.ts:13` |
| `case-documents` Storage bucket | `supabase/migrations/003_document_storage.sql` |
| "Clinical Data" sidebar tab (disabled) | `src/components/patients/case-sidebar.tsx:44` |

**What does NOT exist:**
- Claude API integration / `@anthropic-ai/sdk` dependency
- `mri_extractions` database table
- Extraction logic or server actions
- Clinical data page route or UI components

### Key Discoveries:
- `saveDocumentMetadata()` at `src/actions/documents.ts:82` returns the inserted document row with `id` — we can use this to trigger extraction
- `getDocumentPreviewUrl()` at `src/actions/documents.ts:149` creates signed URLs for inline viewing (no download flag)
- Upload sheet processes files serially in a `for...of` loop at `src/components/documents/upload-sheet.tsx:102` — extraction call fits naturally after `saveDocumentMetadata()` succeeds
- The existing `PdfPreview` component is dialog-based (`Dialog` wrapper). We need a new inline variant for the split-view
- All server actions follow the pattern: `'use server'` → `createClient()` → Zod safeParse → DB operation → `revalidatePath()` → return `{ data }` or `{ error }`
- All tables use `uuid_generate_v4()` PKs, `created_at`/`updated_at`/`deleted_at` soft-delete, `created_by`/`updated_by` audit fields
- RLS is simple: `auth.role() = 'authenticated'` full access
- Only two env vars exist (both `NEXT_PUBLIC_`). `ANTHROPIC_API_KEY` will be the first server-only env var
- Forms use `react-hook-form` + `zodResolver` + shadcn `Form`/`FormField` components + `sonner` toasts

## Desired End State

After implementation:
1. Uploading an MRI report PDF triggers automatic extraction via Claude API
2. Extraction results are stored in `mri_extractions` table with structured data
3. The "Clinical Data" sidebar tab is enabled and links to `/patients/[caseId]/clinical`
4. The clinical data page shows a list of MRI extractions for the case
5. Clicking an extraction opens a split-view: PDF on left, editable extracted data on right
6. Provider can approve, edit + save, or reject each extraction
7. Provider edits are stored in `provider_overrides` JSONB, preserving original AI output
8. Extraction failures show a manual entry fallback

### How to verify:
1. Upload a PDF with document type "MRI Report" → extraction runs automatically, toast shows success with link
2. Navigate to Clinical Data tab → extraction appears in list with confidence badge
3. Click extraction → split-view shows PDF alongside extracted fields
4. Edit a field and save → `provider_overrides` contains the edit, `review_status` = `'edited'`
5. Approve an extraction → `review_status` = `'approved'`
6. Re-extract → old record soft-deleted, new extraction created

## What We're NOT Doing

- Chiropractor report extraction (Story 2.2+)
- Async extraction with realtime subscriptions (sync is fine for MVP)
- Claude Files API (beta, not ZDR-eligible)
- Production HIPAA/BAA setup (synthetic data only for now)
- Granular rate limiting on Claude API calls
- Extraction version history UI (soft-deleted records exist in DB but are not surfaced)

## Implementation Approach

Build bottom-up: database schema → extraction pipeline → server actions → UI components → page route. Each phase is independently testable.

```typescript
import Anthropic from '@anthropic-ai/sdk'
import { mriExtractionResultSchema, type MriExtractionResult } from '@/lib/validations/mri-extraction'

const anthropic = new Anthropic() // reads ANTHROPIC_API_KEY from env

const EXTRACTION_PROMPT = `You are a medical data extraction assistant for a personal injury clinic.
Extract structured information from the attached MRI radiology report.

<instructions>
- Extract the body region scanned (e.g., "Lumbar Spine", "Cervical Spine")
- Extract the MRI study date if present
- Extract each disc level or anatomical finding individually -- do NOT combine
- For each finding, identify the spinal level (e.g., L4-L5) or anatomical location
- Include the radiologist's Impression section verbatim if present
- If a field cannot be determined, return null -- do NOT guess
- Set confidence to "low" if document quality is poor or report is incomplete
- Add extraction_notes for anything ambiguous or missing
</instructions>

Extract the structured data from this MRI report now.`

const EXTRACTION_SCHEMA = {
  type: 'object' as const,
  properties: {
    body_region: {
      type: 'string' as const,
      description: "e.g. 'Lumbar Spine', 'Cervical Spine', 'Right Knee'",
    },
    mri_date: {
      type: ['string', 'null'] as const,
      description: 'ISO 8601 date (YYYY-MM-DD) or null if not found',
    },
    findings: {
      type: 'array' as const,
      items: {
        type: 'object' as const,
        properties: {
          level: { type: 'string' as const },
          description: { type: 'string' as const },
          severity: {
            type: ['string', 'null'] as const,
            enum: ['mild', 'moderate', 'severe', null],
          },
        },
        required: ['level', 'description', 'severity'],
      },
    },
    impression_summary: {
      type: ['string', 'null'] as const,
      description: "Radiologist's impression/conclusion section",
    },
    confidence: {
      type: 'string' as const,
      enum: ['high', 'medium', 'low'],
    },
    extraction_notes: {
      type: ['string', 'null'] as const,
      description: 'Ambiguities, missing data, or quality issues',
    },
  },
  required: ['body_region', 'mri_date', 'findings', 'impression_summary', 'confidence', 'extraction_notes'],
}

export async function extractMriFromPdf(pdfBase64: string): Promise<{
  data?: MriExtractionResult
  rawResponse?: unknown
  error?: string
}> {
  try {
    const response = await anthropic.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 2048,
      messages: [{
        role: 'user',
        content: [
          {
            type: 'document',
            source: { type: 'base64', media_type: 'application/pdf', data: pdfBase64 },
          },
          { type: 'text', text: EXTRACTION_PROMPT },
        ],
      }],
      response_format: {
        type: 'json_schema',
        json_schema: {
          name: 'mri_extraction',
          schema: EXTRACTION_SCHEMA,
          strict: true,
        },
      },
    })

    const textBlock = response.content.find((b) => b.type === 'text')
    if (!textBlock || textBlock.type !== 'text') {
      return { error: 'No text response from Claude' }
    }

    const parsed = JSON.parse(textBlock.text)
    const validated = mriExtractionResultSchema.safeParse(parsed)

    if (!validated.success) {
      return { error: 'Extraction output failed validation', rawResponse: parsed }
    }

    return { data: validated.data, rawResponse: parsed }
  } catch (err) {
    return { error: err instanceof Error ? err.message : 'Claude API call failed' }
  }
}
```

#### 2. Server Actions
**File**: `src/actions/mri-extractions.ts` (new)

```typescript
'use server'

import { createClient } from '@/lib/supabase/server'
import { revalidatePath } from 'next/cache'
import { extractMriFromPdf } from '@/lib/claude/extract-mri'
import type { MriReviewFormValues } from '@/lib/validations/mri-extraction'

// --- Trigger extraction for a document ---

export async function extractMriReport(documentId: string) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  // Get document metadata
  const { data: doc, error: docError } = await supabase
    .from('documents')
    .select('id, case_id, file_path, document_type')
    .eq('id', documentId)
    .is('deleted_at', null)
    .single()

  if (docError || !doc) return { error: 'Document not found' }
  if (doc.document_type !== 'mri_report') return { error: 'Not an MRI report' }

  // Soft-delete any existing extraction for this document
  await supabase
    .from('mri_extractions')
    .update({ deleted_at: new Date().toISOString(), updated_by_user_id: user.id })
    .eq('document_id', documentId)
    .is('deleted_at', null)

  // Create pending extraction record
  const { data: extraction, error: insertError } = await supabase
    .from('mri_extractions')
    .insert({
      document_id: documentId,
      case_id: doc.case_id,
      extraction_status: 'processing',
      extraction_attempts: 1,
      created_by_user_id: user.id,
      updated_by_user_id: user.id,
    })
    .select()
    .single()

  if (insertError || !extraction) return { error: 'Failed to create extraction record' }

  // Download PDF from Supabase Storage
  const { data: fileData, error: downloadError } = await supabase.storage
    .from('case-documents')
    .download(doc.file_path)

  if (downloadError || !fileData) {
    await supabase.from('mri_extractions').update({
      extraction_status: 'failed',
      extraction_error: 'Failed to download PDF from storage',
      updated_by_user_id: user.id,
    }).eq('id', extraction.id)
    return { error: 'Failed to download PDF' }
  }

  // Check for encrypted PDF
  const firstBytes = new Uint8Array(await fileData.slice(0, 8192).arrayBuffer())
  const headerStr = new TextDecoder().decode(firstBytes)
  if (headerStr.includes('/Encrypt')) {
    await supabase.from('mri_extractions').update({
      extraction_status: 'failed',
      extraction_error: 'PDF is encrypted/password-protected',
      updated_by_user_id: user.id,
    }).eq('id', extraction.id)
    return { error: 'PDF is encrypted. Please upload an unprotected version.' }
  }

  // Convert to base64 and call Claude
  const arrayBuffer = await fileData.arrayBuffer()
  const pdfBase64 = Buffer.from(arrayBuffer).toString('base64')

  const result = await extractMriFromPdf(pdfBase64)

  if (result.error || !result.data) {
    // Retry once on failure
    const retry = await extractMriFromPdf(pdfBase64)

    if (retry.error || !retry.data) {
      await supabase.from('mri_extractions').update({
        extraction_status: 'failed',
        extraction_error: retry.error ?? result.error ?? 'Extraction failed',
        extraction_attempts: 2,
        raw_ai_response: retry.rawResponse ?? result.rawResponse ?? null,
        updated_by_user_id: user.id,
      }).eq('id', extraction.id)
      revalidatePath(`/patients/${doc.case_id}/clinical`)
      return { error: retry.error ?? result.error }
    }

    // Retry succeeded
    await updateExtractionSuccess(supabase, extraction.id, retry, user.id, 2)
    revalidatePath(`/patients/${doc.case_id}/clinical`)
    return { data: { extractionId: extraction.id } }
  }

  // First attempt succeeded
  await updateExtractionSuccess(supabase, extraction.id, result, user.id, 1)
  revalidatePath(`/patients/${doc.case_id}/clinical`)
  revalidatePath(`/patients/${doc.case_id}/documents`)
  return { data: { extractionId: extraction.id } }
}

async function updateExtractionSuccess(
  supabase: Awaited<ReturnType<typeof createClient>>,
  extractionId: string,
  result: { data?: { body_region: string; mri_date: string | null; findings: unknown; impression_summary: string | null; confidence: string; extraction_notes: string | null }; rawResponse?: unknown },
  userId: string,
  attempts: number,
) {
  const data = result.data!
  await supabase.from('mri_extractions').update({
    extraction_status: 'completed',
    body_region: data.body_region,
    mri_date: data.mri_date,
    findings: data.findings,
    impression_summary: data.impression_summary,
    ai_model: 'claude-sonnet-4-6',
    ai_confidence: data.confidence,
    extraction_notes: data.extraction_notes,
    raw_ai_response: result.rawResponse ?? null,
    extraction_attempts: attempts,
    extracted_at: new Date().toISOString(),
    updated_by_user_id: userId,
  }).eq('id', extractionId)
}

// --- List extractions for a case ---

export async function listMriExtractions(caseId: string) {
  const supabase = await createClient()

  const { data, error } = await supabase
    .from('mri_extractions')
    .select('*, document:documents!document_id(file_name, file_path)')
    .eq('case_id', caseId)
    .is('deleted_at', null)
    .order('created_at', { ascending: false })

  if (error) return { error: error.message, data: [] }
  return { data: data ?? [] }
}

// --- Get single extraction ---

export async function getMriExtraction(extractionId: string) {
  const supabase = await createClient()

  const { data, error } = await supabase
    .from('mri_extractions')
    .select('*, document:documents!document_id(file_name, file_path)')
    .eq('id', extractionId)
    .is('deleted_at', null)
    .single()

  if (error) return { error: error.message }
  return { data }
}

// --- Approve extraction (no edits) ---

export async function approveMriExtraction(extractionId: string) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  const { data, error } = await supabase
    .from('mri_extractions')
    .update({
      review_status: 'approved',
      reviewed_by_user_id: user.id,
      reviewed_at: new Date().toISOString(),
      updated_by_user_id: user.id,
    })
    .eq('id', extractionId)
    .is('deleted_at', null)
    .select('case_id')
    .single()

  if (error) return { error: error.message }
  revalidatePath(`/patients/${data.case_id}/clinical`)
  return { data }
}

// --- Save edits and approve ---

export async function saveAndApproveMriExtraction(
  extractionId: string,
  overrides: MriReviewFormValues,
) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  const { data, error } = await supabase
    .from('mri_extractions')
    .update({
      review_status: 'edited',
      provider_overrides: overrides,
      reviewed_by_user_id: user.id,
      reviewed_at: new Date().toISOString(),
      updated_by_user_id: user.id,
    })
    .eq('id', extractionId)
    .is('deleted_at', null)
    .select('case_id')
    .single()

  if (error) return { error: error.message }
  revalidatePath(`/patients/${data.case_id}/clinical`)
  return { data }
}

// --- Reject extraction ---

export async function rejectMriExtraction(extractionId: string, reason: string) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { error: 'Not authenticated' }

  const { data, error } = await supabase
    .from('mri_extractions')
    .update({
      review_status: 'rejected',
      extraction_notes: reason,
      reviewed_by_user_id: user.id,
      reviewed_at: new Date().toISOString(),
      updated_by_user_id: user.id,
    })
    .eq('id', extractionId)
    .is('deleted_at', null)
    .select('case_id')
    .single()

  if (error) return { error: error.message }
  revalidatePath(`/patients/${data.case_id}/clinical`)
  return { data }
}
```

## Performance Considerations

- Extraction is synchronous (~5-15 seconds). Acceptable for MVP with loading spinner.
- PDF download from Supabase Storage is server-to-server (fast).
- Base64 encoding doubles file size in memory — for a 10MB PDF, ~20MB RAM. Fine for serverless.
- Signed URLs expire in 1 hour — sufficient for a review session.
