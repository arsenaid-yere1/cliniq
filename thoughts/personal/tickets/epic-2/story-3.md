# User Story 2.3 — Clinical Case Summary Generation

**As a** provider
**I want** a summarized clinical case view
**So that** I can quickly assess patient status.

## Acceptance Criteria

- [ ] Summary includes:
  - [ ] Chief complaint
  - [ ] Imaging findings
  - [ ] Prior treatment
  - [ ] Symptoms timeline
  - [ ] Suggested diagnoses
- [ ] Provider can edit summary fields
